# راهنمای تست ایجاد سفارش جدید

## 🎯 هدف
تست کامل ایجاد سفارش جدید با یکپارچگی Frontend-Backend و ذخیره در دیتابیس Django.

## 🚀 مراحل تست

### مرحله 1: اطمینان از اجرای Backend
```bash
cd backend
python manage.py runserver 8000
```

**بررسی**: Django server باید روی `http://127.0.0.1:8000` در حال اجرا باشد.

### مرحله 2: تست API Connection
1. فایل `api-test.html` را در مرورگر باز کنید
2. روی دکمه "تست احراز هویت" کلیک کنید
3. اطمینان حاصل کنید که ورود با `manager/123456` موفق است

### مرحله 3: تست ایجاد سفارش (روش اول - صفحه تست)
1. فایل `order-creation-test.html` را در مرورگر باز کنید
2. فرم را با اطلاعات زیر پر کنید:
   - **نام دانشجو**: احمد محمدی
   - **دانشگاه**: دانشگاه قم
   - **رشته**: حقوق محض
   - **مقطع**: ارشد
   - **نوع سفارش**: نوشتن رساله
   - **مهلت**: تاریخ آینده (مثلاً 2026-06-01)
   - **توضیحات**: رساله در زمینه حقوق بین‌الملل
3. روی "ثبت سفارش" کلیک کنید
4. بررسی کنید که پیام موفقیت نمایش داده شود

### مرحله 4: تست ایجاد سفارش (روش دوم - سیستم اصلی)
1. فایل `index.html` را در مرورگر باز کنید
2. اطمینان حاصل کنید که نقش "مدیر" انتخاب شده است
3. روی دکمه "+" در footer یا "سفارش جدید" در داشبورد کلیک کنید
4. فرم modal را پر کنید
5. سفارش را ثبت کنید

### مرحله 5: بررسی در Django Admin
1. به `http://127.0.0.1:8000/admin/` بروید
2. با `admin/admin` وارد شوید (یا superuser ایجاد کنید)
3. بخش "Orders" را بررسی کنید
4. سفارش جدید باید در لیست نمایش داده شود

### مرحله 6: بررسی در Frontend
1. صفحه را refresh کنید
2. به بخش "سفارشات" بروید
3. سفارش جدید باید در لیست نمایش داده شود

## 📊 نتایج مورد انتظار

### ✅ موفقیت‌آمیز
- پیام "سفارش با موفقیت در سیستم ثبت شد" نمایش داده شود
- سفارش در Django Admin Panel نمایش داده شود
- سفارش در لیست سفارشات Frontend نمایش داده شود
- Student Profile جدید ایجاد شود
- آمار داشبورد به‌روزرسانی شود

### ❌ خطاهای احتمالی
- **"API در دسترس نیست"**: Django server اجرا نشده
- **"خطا در ثبت سفارش"**: مشکل در validation یا database
- **"اتصال رد شد"**: مشکل CORS یا authentication

## 🔧 عیب‌یابی

### Django Server اجرا نمی‌شود
```bash
cd backend
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver 8000
```

### CORS Errors
در `backend/config/settings.py` بررسی کنید:
```python
CORS_ALLOW_ALL_ORIGINS = True  # فقط برای development
```

### Database Errors
```bash
python manage.py makemigrations
python manage.py migrate
python manage.py create_sample_data
```

## 🎯 بررسی‌های نهایی

### در Django Admin
1. **Students**: پروفایل دانشجوی جدید ایجاد شده
2. **Orders**: سفارش با وضعیت "pending" ثبت شده
3. **Users**: کاربر دانشجو (اگر نیاز باشد) ایجاد شده

1. **Dashboard**: آمار به‌روزرسانی شده (تعداد سفارشات)

## 🏆 تایید موفقیت

اگر تمام مراحل بالا موفق باشند، یکپارچگی کامل Frontend-Backend تایید می‌شود:

- ✅ Frontend می‌تواند با Django API ارتباط برقرار کند
- ✅ داده‌ها در SQLite database ذخیره می‌شوند
- ✅ Real-time updates در Frontend کار می‌کند
- ✅ سیستم آماده برای استفاده واقعی است
