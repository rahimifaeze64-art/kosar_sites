# راهنمای گردش دفاع و فارغ‌التحصیلی دانشجو

## نحوه دسترسی

1. به عنوان کارمند وارد سیستم شوید
2. از منوی سمت راست، گزینه "دانشجویان" را انتخاب کنید
3. روی دکمه "ویرایش پروفایل" دانشجوی مورد نظر کلیک کنید
4. از تب‌های بالای مودال، "گردش دفاع" یا "گردش فارغ‌التحصیلی" را انتخاب کنید

## گردش دفاع

### مراحل پیش‌فرض:

1. **ثبت درخواست دفاع**
   - دانشجو درخواست دفاع از رساله را ثبت می‌کند

2. **تایید استاد راهنما**
   - استاد راهنما آمادگی دانشجو را تایید می‌کند

3. **تعیین اساتید داور**
   - اساتید داور برای بررسی رساله انتخاب می‌شوند

4. **ارسال رساله به داوران**
   - رساله به اساتید داور ارسال می‌شود

5. **دریافت نظرات داوران**
   - نظرات و پیشنهادات داوران دریافت می‌شود

6. **تعیین تاریخ دفاع**
   - تاریخ و زمان جلسه دفاع مشخص می‌شود

7. **برگزاری جلسه دفاع**
   - جلسه دفاع از رساله برگزار می‌شود

8. **اعلام نتیجه دفاع**
   - نتیجه نهایی دفاع اعلام می‌شود

## گردش فارغ‌التحصیلی

### مراحل پیش‌فرض:

1. **تکمیل واحدهای درسی**
   - تمام واحدهای درسی مورد نیاز تکمیل شود

2. **تایید نهایی رساله**
   - رساله نهایی توسط استاد راهنما تایید شود

3. **ثبت درخواست فارغ‌التحصیلی**
   - دانشجو درخواست فارغ‌التحصیلی را ثبت کند

4. **بررسی مدارک توسط آموزش**
   - واحد آموزش مدارک را بررسی و تایید کند

5. **تسویه حساب مالی**
   - تسویه حساب مالی با دانشگاه انجام شود

6. **تسویه حساب کتابخانه**
   - تسویه حساب با کتابخانه انجام شود

7. **صدور گواهی موقت**
   - گواهی موقت فارغ‌التحصیلی صادر شود

8. **صدور مدرک فارغ‌التحصیلی**
   - مدرک رسمی فارغ‌التحصیلی صادر شود

## نحوه استفاده

### تکمیل مراحل:

1. **کلیک روی دکمه مرحله**
   - روی دکمه دایره‌ای شکل هر مرحله کلیک کنید
   - مرحله به حالت "تکمیل شده" تغییر می‌کند
   - تیک سبز نمایش داده می‌شود
   - تاریخ تکمیل به صورت خودکار ثبت می‌شود

2. **لغو تکمیل مرحله**
   - دوباره روی دکمه مرحله کلیک کنید
   - مرحله به حالت "در انتظار" برمی‌گردد

### ویژگی‌های نمایشی:

✅ **نوار پیشرفت**
- نمایش درصد پیشرفت کلی
- رنگ آبی برای گردش دفاع
- رنگ سبز برای گردش فارغ‌التحصیلی

✅ **شمارنده مراحل**
- نمایش تعداد مراحل تکمیل شده از کل مراحل

✅ **وضعیت مراحل**
- تیک سبز: مرحله تکمیل شده
- شماره: مرحله در انتظار
- برچسب رنگی: وضعیت فعلی

✅ **تاریخ تکمیل**
- ثبت خودکار تاریخ تکمیل هر مرحله
- نمایش تاریخ به شمسی

## ذخیره‌سازی

- تمام تغییرات به صورت خودکار در `localStorage` ذخیره می‌شوند
- کلید ذخیره‌سازی: `students_data`
- هر دانشجو دارای دو آرایه است:
  - `defenseWorkflow`: مراحل گردش دفاع
  - `graduationWorkflow`: مراحل گردش فارغ‌التحصیلی

## ساختار داده

```javascript
{
  defenseWorkflow: [
    {
      name: "نام مرحله",
      completed: true/false,
      completedAt: "2024-01-15T10:30:00.000Z" // یا null
    }
  ],
  graduationWorkflow: [
    // مشابه defenseWorkflow
  ]
}
```

## نکات مهم

⚠️ **ترتیب مراحل**: مراحل به ترتیب منطقی چیده شده‌اند اما می‌توان هر مرحله را مستقل تکمیل کرد

⚠️ **ذخیره تغییرات**: تغییرات هر مرحله بلافاصله ذخیره می‌شود

⚠️ **تاریخ تکمیل**: تاریخ تکمیل به صورت خودکار با تاریخ سیستم ثبت می‌شود

⚠️ **مراحل پیش‌فرض**: اگر دانشجو مراحل تعریف شده نداشته باشد، مراحل پیش‌فرض ایجاد می‌شوند

## مزایا

✨ **پیگیری آسان**: پیگیری وضعیت هر دانشجو در یک نگاه

✨ **شفافیت**: مشخص بودن مراحل باقی‌مانده

✨ **گزارش‌گیری**: امکان محاسبه درصد پیشرفت

✨ **تاریخچه**: ثبت تاریخ تکمیل هر مرحله

✨ **رابط کاربری ساده**: تکمیل مراحل با یک کلیک
