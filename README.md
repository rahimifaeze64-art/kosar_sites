# سیستم مدیریت تحصیلی دانشجویان بین‌المللی

## توضیحات پروژه

این پروتوتایپ یک سیستم کامل و ماژولار برای مدیریت فرآیندهای تحصیلی دانشجویان بین‌المللی است که شامل تمام مراحل از ثبت‌نام تا فارغ‌التحصیلی می‌باشد. سیستم با معماری PWA طراحی شده و قابلیت تبدیل به اپلیکیشن موبایل را دارد.

## نقش‌های سیستم

### 1. مدیر (Manager)
- تایید سفارشات جدید
- مدیریت کاربران
- نظارت بر کل فرآیند

### 2. کارمند (employee)
- تخصیص وظایف به عاملها
- بررسی و تایید/رد کارها
- پیگیری مراحل مختلف

### 3. عامل (Doctor)
- نوشتن مقالات و رساله‌ها
- ارائه عناوین پیشنهادی
- انجام تعدیلات مورد نیاز

### 4. دانشجو (Student)
- ثبت سفارش جدید
- پیگیری وضعیت سفارشات
- مشاهده پیشرفت کار

### 5. مترجم (Translator)
- ترجمه رساله‌ها
- تلخیص متون
- ترجمه اسناد

## مراحل فرآیند

### مرحله 1: ثبت سفارش
- دانشجو سفارش را ثبت می‌کند
- مدیر سفارش را بررسی و تایید می‌کند

### مرحله 2: تخصیص وظیفه
- کارمند وظیفه را به عامل مناسب تخصیص می‌دهد
- عامل شروع به کار می‌کند

### مرحله 3: بررسی و تایید
- کارمند کار انجام شده را بررسی می‌کند
- در صورت تایید، به مرحله بعد می‌رود
- در صورت رد، به عامل برای اصلاح برمی‌گردد

### مرحله 4: تکمیل
- پس از تایید نهایی، کار تکمیل می‌شود

## ویژگی‌های کلیدی

### 🔐 سیستم احراز هویت
- ورود با نام کاربری و رمز عبور
- مدیریت نقش‌های مختلف کاربران
- امکان ثبت نام دانشجویان جدید

### 💰 مدیریت مالی
- محاسبه خودکار هزینه‌ها بر اساس نوع سفارش
- تعیین سهم عامل و مدیر
- پیگیری وضعیت پرداخت

### 👥 مدیریت کاربران (مدیر)
- افزودن کاربران جدید
- تعیین نقش و دسترسی‌ها
- فعال/غیرفعال کردن کاربران

### 📊 داشبورد اختصاصی
- داشبورد مخصوص هر نقش
- آمار و گزارش‌های لحظه‌ای
- فعالیت‌های اخیر

### 📱 PWA (Progressive Web App)
- قابلیت نصب روی موبایل
- کار آفلاین
- نوتیفیکیشن‌ها

### 🔄 تبدیل نقش
- امکان تغییر نقش از طریق سوییچ در هدر
- نمایش محتوای مخصوص هر نقش

### 💾 ذخیره‌سازی موقت
- تمام داده‌ها در localStorage ذخیره می‌شود
- امکان بازیابی اطلاعات پس از بستن مرورگر

### 📱 طراحی ریسپانسیو
- سازگار با موبایل و تبلت
- فوتر نویگیشن مشابه اپ‌های موبایل
- استفاده از Tailwind CSS

### 🌐 پشتیبانی از RTL
- طراحی مناسب برای زبان فارسی و عربی
- فونت Vazirmatn

## ساختار پروژه

```
├── index.html              # صفحه اصلی
├── manifest.json           # فایل PWA
├── sw.js                  # Service Worker
├── assets/
│   └── css/
│       └── styles.css     # استایل‌های اصلی
├── js/
│   ├── config.js          # تنظیمات و ثوابت
│   ├── auth.js            # احراز هویت
│   ├── data.js            # مدیریت داده‌ها
│   ├── dashboard.js       # داشبورد نقش‌ها
│   ├── orders.js          # مدیریت سفارشات
│   ├── users.js           # مدیریت کاربران
│   ├── profile.js         # مدیریت پروفایل
│   ├── modals.js          # مدیریت مودال‌ها
│   └── app.js             # کنترلر اصلی
├── server.py              # سرور تست محلی
├── start-server.bat       # اجرای سرور در ویندوز
└── README.md              # این فایل
```

## نحوه اجرا

### روش 1: سرور Python
1. فایل‌ها را در یک پوشه قرار دهید
2. دستور زیر را اجرا کنید:
   ```bash
   python server.py
   ```
3. مرورگر خودکار باز می‌شود

### روش 2: ویندوز
1. فایل `start-server.bat` را دابل کلیک کنید
2. مرورگر خودکار باز می‌شود

### روش 3: سرور محلی دیگر
1. هر سرور HTTP محلی را اجرا کنید
2. فایل `index.html` را در مرورگر باز کنید

## تست سیستم

1. با اکانت مدیر وارد شوید: `manager` / `123456`
2. از سوییچ نقش در بالای صفحه استفاده کنید
3. هر نقش را تست کنید:
   - **مدیر**: مدیریت کاربران، سفارشات، گزارشات
   - **دانشجو**: ثبت سفارش، پیگیری وضعیت
   - **عامل**: مشاهده وظایف محول شده
   - **کارمند**: تخصیص وظایف، تایید/رد
   - **مترجم**: وظایف ترجمه

## تکنولوژی‌های استفاده شده

- **HTML5**: ساختار صفحه
- **Tailwind CSS**: طراحی و استایل
- **Alpine.js**: منطق فرانت‌اند
- **Font Awesome**: آیکون‌ها
- **Vazirmatn Font**: فونت فارسی

## داده‌های نمونه

### کاربران پیش‌فرض:
- **مدیر**: username: `manager`, password: `123456`
- **کارمند**: username: `employee1`, password: `123456`
- **عامل**: username: `doctor1`, password: `123456`
- **مترجم**: username: `translator1`, password: `123456`
- **دانشجویان**: 15 دانشجوی عراقی با نام‌های واقعی

### نمونه دانشجویان:
- ماجد عبدالرحمن صالح (جامعة بغداد)
- قاسم محمد علي (الجامعة المستنصرية)
- محمد احمد حسين (جامعة البصرة)
- احمد علي محمود (جامعة الموصل)
- و 11 دانشجوی دیگر...

### سفارشات نمونه:
- سفارشات در مراحل مختلف (در انتظار، در حال انجام، تکمیل شده، رد شده)
- محاسبه هزینه و سهم‌بندی
- تاریخچه رد و تایید

## نکات مهم

- این یک پروتوتایپ است و برای استفاده واقعی نیاز به backend دارد
- داده‌ها فقط در مرورگر ذخیره می‌شود
- برای تست کامل، نقش‌های مختلف را امتحان کنید
